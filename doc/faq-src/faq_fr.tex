\section{Installation}
\subsection{Ou trouver un visioneur de fax pour Windows?}

Vous pouvez utiliser n'importe quel programme qui peut afficher les fichier TIFF,  cependant
certains affichent les faxs avec une basse résolution avec une (demi) taille incorrecte.
some of them display faxes in low resolution with an incorrect (half) height.

Heureusement, tous les systèmes Windows récents sont livrés avec un programme qui le fait correctement:

\begin{description}
\item[Windows 95/98/ME/2000:]
Vous pouvez utiliser l'application Imaging du menu Accessoires.
      Pour l'utiliser avec YaHFC, cliquez sur le bouton de parcours des fichiers \texttt{Ligne de commande pour visualiser les fax}
      pour trouver le programme \texttt{kodakimg.exe} sur votre disque dur.\\
      Ce fichier est habituellement dasn le répertoire Windows (Win 98/ME) \emph{ou} \\
      dans \verb.Program Files\Windows NT\Accessoires\ImageVue. (Win 2k).

\item[Windows XP:]
Vous pouvez utiliser le "visualiseur d'images et de fax" intégré (Désolé, Je ne connais pas son emplacement exact).
      Pour l'utiliser,entrez le texte suivant dans le champ \texttt{Ligne de commande pour visualiser les fax} boite:\\
      \verb#rundll32.exe shimgvw.dll,ImageView_Fullscreen %s#
 \end{description}


\subsection{Ou Trouver un visualiseur de fax pour Linux/*BSD/... ?}
\texttt{kfax} fonctionne très bien pour moi, mais comme pour Windows, vous pouvez utiliser n'importe quel programme
capable d'afficher les fichiersTIFF , cherchez dans la base des paquets de votre distribution
(xloadimage ne fonctionne cependant \emph{pas} pour moi).


\subsection{Que devrai je utiliser comme visualiseur sur  Mac OS X?}
Entrez seulement \verb.open %s. dans le champ \texttt{Ligne de commande pour visualiser les fax} .
Le faxs devrait automatiquement s'ouvrir avec l'application par défaut.
Merci à Scott Harris pour cette information.


\subsection{Ou trouver un visualiseur PostsScript?}
\begin{description}
\item[Windows:] Utilisez Ghostview depuis \url{http://www.cs.wisc.edu/~ghost/}
\item[Linux/*BSD/...:] Installez seulement un paquet Visualiseur Postscript
                (i.e. \texttt{gv, kghostview, gnome-gv, ...})
\end{description}

\section{Utilisation du programme}

\subsection{Comment éditer les modèles de page de garde?}
Les modèles doivent être au même format Postscript que ceux utilisés par le programme HylaFAX
\texttt{faxcover} . Lisez les pages suivantes pour voir comment  créer/éditer
de tels fichiers: \\
\url{http://www.hylafax.org/HylaFAQ/Q202.html}\\
\url{http://www.hylafax.org/howto/tweaking.html}\\

Sinon, Depuis la version 0.3.7, vous pouvez utiliser des page de garde en HTML ou, avec un module (plugin), en XSL:FO ou format ODT (texte Open Document).

\subsection{J'ai cree un document/page de couverture HTMLI created a HTML mais la mise en forme dans YajHFC parait mauvaise!}


YajHFC utilise le support HTML intégré de Java (\texttt{HTMLEditorKit} / \texttt{HTMLDocument}) pour convertir le HTML en PostScript. Ce support est limite, aussi, surtout il ne supporte que la version HTML 3.2.\\
Cela signifie que les mises en forme complexes seront souvent mal affichées dans YajHFC.
Pour créer la mise en page que vous désirez, vous aurez de base les alternatives suivantes:


\begin{itemize}
 \item Faire ``des essais \& erreurs'' jusqu'a ce que la mise en page apparaisse correctement (Le bouton Prévisualiser dans la boite de dialogue d'envoi affichera la mise en forme de la page HTML convertie).
 \item Utilisez un éditeur HTML comme Ekit (\url{http://www.hexidec.com/ekit.php}) Qui utilise aussi un support Java, Aussi il affichera la page comme YajHFC.
 \item Utilisez un autre format pour la page de garde (comme XSL:FO ou ODT avec le module FOP).
\end{itemize}

\subsection{Quels champs sont reconnus dans une page de couverture HTML?}

les mot suivants``words'' sont remplacés (sans distinction de casse) par les valeurs correspondantes quand une page HTML est utilisée comme page de couverture:

\begin{center}
\begin{tabular}{|l|l|}
\hline
\bfseries Word & \bfseries Meaning \\
\hline\hline
\ttfamily @@Name@@ & Nom du récipiendaire \\\hline
\ttfamily @@Location@@ & Adresse du récipiendaire \\\hline
\ttfamily @@Company@@ & Societe du récipiendaire \\\hline
\ttfamily @@Faxnumber@@ & Numero de fax du récipiendaire \\\hline
\ttfamily @@Voicenumber@@ & numero de telephone du récipiendaire \\\hline
\ttfamily @@FromName@@ & Nom de l'expéditeur \\\hline
\ttfamily @@FromLocation@@ & Adresse de l'expéditeur \\\hline
\ttfamily @@FromCompany@@ & Societe de l'expéditeur \\\hline
\ttfamily @@FromFaxnumber@@ & Numero de fax de l'expéditeur \\\hline
\ttfamily @@FromVoicenumber@@ & Numero de téléphone de l'expéditeur \\\hline
\ttfamily @@FromEMail@@ & Adresse e-mail de l'expéditeur \\\hline
\ttfamily @@Subject@@ & Sujet du fax \\\hline
\ttfamily @@Date@@ & Date du jour \\\hline
\ttfamily @@PageCount@@ & Le nombre de pages en excluant la page de garde \\\hline
\ttfamily @@Comments@@ & Les commentaires pour ce Fax \\\hline
\end{tabular}
\end{center}

Ces remplacements sont fais sur le code source de la page, aussi ces mots ne seront certainement pas reconnus s'il y a des chaine de formatatge dans l'un d'eux (exemple. \texttt{@@sub\textit{ject@@}}).

\subsection{Je preferais l'ancienne boite de dialogue d'envoi. Puis je la conserver?}

Ouvrez la boite de dialogue des Options et selectionnez \texttt{Traditionnel} comme ``Style de boite d'envoi''.


\subsection{Je voudrais accéder à un annuaire téléphonique au travers de JDBC, maist YajHFC ne trouve pas les drivers même si je spécifie un  "class path" correct au chargement de la machine virtuelle java.}

Si vous utilisez l'argument de ligne de commande \texttt{-jar} , java ignore les "class path" utilisateur.
Aussi,Démarrez YajHFC en utilisant les commandes suivantes (remplacez \texttt{/path/to/dbdriver.jar} et \texttt{/path/to/yajhfc.jar} par les répertoires et nom des fichiers respectifs réels bien sur):
\begin{description}
\item [Linux/Unix:] \verb#java -classpath /path/to/dbdriver.jar:/path/to/yajhfc.jar yajhfc.Launcher#
\item [Windows:] \verb#java -classpath c:\path\to\dbdriver.jar;c:\path\to\yajhfc.jar yajhfc.Launcher#
\end{description}

\subsection{Quelles valeurs puis je entrer comme \texttt{matches} opérateurs dans le champ des filtres personnalisés?}

Expressions régulières, Un court manuel de référence sur les syntaxes acceptées se trouve à l'adresse :
\url{http://java.sun.com/j2se/1.5.0/docs/api/java/util/regex/Pattern.html}

Notez que les expressions régulières ne sont pas identiques aux "wildcards :
Par exemple, pour avoir l'effet de \verb.*. "wildcard", vous devez utiliser \verb#.*# and
pour l'effet du "wildcard" \verb#?# use \verb#.#.

\subsection{Quels arguments de ligne de commande YajHFC accepte ?}

\begin{verbatim}
Usage Général :
java -jar yajhfc.jar [--help] [--debug] [--admin] [--background|--noclose]
         [--configdir=directory] [--loadplugin=filename] [--logfile=filename]
         [--showtab=0|R|1|S|2|T] [--recipient=...] [--stdin | filename ...]
Description des Arguments :
filename     Un ou plusieurs nom de fichier à envoyer.
--stdin      Lit le fichier depuis l'entrée standard
--recipient  Spécifie le numéro de téléphone du récipiendaire auquel envoyer le fax.
             Vous devez spécifier plusieurs arguments pour des recipiendaires multiples.
--admin      Démarre en mode Administrateur
--debug      Affiche des information de deboggage.
--logfile    Le fichier de log pour les informations de debogage  (Si rien n'est spécifié, la sortie standard est utilisée)
--background S'il n'y a pas d'instance en cours, démarre une nouvelle instance
             et la termine (après avoir soumis le fichier à envoyer)
--noclose    Ne pas fermer YajHFC après la soumission du fax
--showtab    Spécifie l'onglet à afficher au démarrage. Spécifiez 0 ou R pour l'onglet "Reçus",
             1 ou S pour l'onglet "Envoyés" ou 2 ou T pour l'onglet "Transmission".
--loadplugin Spécifie le fichier jar de plugin YajHFC à charger
--no-plugins Supprime le chargement des modules (plugins) depuis le fichier plugin.lst.
--configdir  Répertoire de configuration à utiliser au lieu de  ~/.yajhfc
--help       Affiche ce texte
\end{verbatim}


\subsection{Quand j'essaie de visualiser les fax envoyés, j'ai toujours un message d'erreur disant
   \texttt{Format de fichier PCL non supporté}, bien que tous les documents soient en PostScript/PDF.}

Cochez la case  \texttt{Utilisez le type de fichier PCL bugfix} dans la boite de dialogue des options et essayez à nouveau.

Certaines version de HylaFAX rapportent de façon incorrecte les fichiers ``PCL'' pour tous les documents
associés à un travail. Si cette case à cocher est active, YajHFC essaie de trouver
le type de fichier PCL s'il est rapporté (Ceci fonctionne en général assez bien).

\subsection{I often get an error when I am sending two or more faxes in a row. What can I do about it?}

Some versions of the HylaFAX server seem to have problems when more than one fax is submitted per session.

To work around that problem, go to the \texttt{Server} tab in the Options dialog, check the \texttt{Create new session for every action} box and try if the problem persists.
If this does \emph{not} help, please mail me a bug report.


\subsection{YajHFC fonctionnant sous windows sauve quelquefois sa configuration dans \texttt{C:\textbackslash .yajhfc} au lieu de \texttt{C:\textbackslash Documents and Settings\textbackslash USERNAME\textbackslash .yajhfc}}

Par défaut YajHFC sauve ses informations de configuration dans le sous répertoire \texttt{.yajhfc} du répertoire retourné par
la propriété système java \texttt{user.home}.
Quelquefois des versions de java semblent ne pas affecter correctement cette variable d'environnement ce qui explique le problème ci dessus.

Comme contournement, vous pouvez affecter cette variable explicitement au lancement de YajHFC en utilisant l'option java \texttt{-D} en ligne de commande, par exemple: \\
\texttt{java -Duser.home=\%USERPROFILE\% -jar "C:\textbackslash Program Files\textbackslash yajhfc.jar"}

\section{Divers}

\subsection{Pourquoi les mots de passe sont il stockés en clair ?}

Simplement car il n'y a pas de méthode beaucoup meilleure.

YajHFC pourrait encoder/``crypter'' les mots de passe avant de les stocker, mais si cela le faisait,
vous pourriez toujours regarder le code source et trouver comment les décrypter
(même si YajHFC était un logiciel fermé (propriétaire) vous pourriez le désassembler
ou expérimenter pour les retrouver ).

La seule méthode sure vous demanderait d'entrer le mot de passe a chaque
exécutions de YajHFC, mais à mon avis cela n'apporterait pas de plus pour
entrer le  ``vrai'' mot de passe.


\subsection{Pourquoi avez vous choisi ce nom stupide ?}

YajHFC a démarré comme un projet de test pour  Java et la librairie \texttt{gnu.hylafax} aussi
je n'avais pas de  ``joli'' nom. Après un moment de travail dessus, j'ai vu que cela devenait
utilisable, j'ai donc choisi de le nommer.
Comme je travaillais aussi sur SuSE  Linux a ce moment je savais qu'il
y avait déjà de nombreux projets clients java hylfax, Je l'ai simplement appellé
``\textbf{y}et \textbf{a}nother \textbf{J}ava \textbf{H}ylaFAX \textbf{c}lient''.
